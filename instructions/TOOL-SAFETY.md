# `instructions/TOOL-SAFETY.md`

이 문서의 규칙은 `AGENTS.md`의 "도구 사용 안전 규칙" 섹션에서 참조된다.

---

## 컨텍스트 부식 인식

- 대화가 10턴 이상 진행되었거나 컨텍스트 압축이 발생한 후에는,
  편집 대상 파일을 반드시 다시 읽는다.
- 자신의 기억에 있는 파일 내용을 신뢰하지 않는다.
  자동 압축이 해당 컨텍스트를 조용히 제거했을 수 있다.
- 이 규칙은 `instructions/SESSIONS.md`의 UPDATES.md stale check와 별개다.
  UPDATES.md는 지침 파일의 신선도를, 이 규칙은 작업 대상 코드 파일의 신선도를 관리한다.
- 이 규칙은 편집 대상 코드 파일에 적용한다.
  `PLANS.md` 재판정은 `instructions/SESSIONS.md`의 별도 트리거를 따른다.

---

## 편집 무결성 주기

- 파일을 편집하기 전에 해당 파일을 반드시 다시 읽는다.
- 편집 후에도 해당 파일을 다시 읽어 변경이 올바르게 적용되었는지 확인한다.
- Edit 도구는 old_string이 현재 파일 내용과 일치하지 않으면 조용히 실패할 수 있다.
- 같은 파일에 3회 이상 연속 편집할 경우 중간에 확인 읽기를 수행한다.

---

## 리팩터링 검색 완전성

- 함수, 타입, 변수의 이름을 변경하거나 삭제할 때
  단일 grep으로 모든 참조를 찾았다고 가정하지 않는다.
- 다음 범주를 각각 별도로 검색한다:
  - 직접 호출 및 참조
  - 타입 수준 참조 (인터페이스, 제네릭)
  - 이름을 포함하는 문자열 리터럴
  - 동적 import 및 require() 호출
  - re-export 및 barrel 파일 항목
  - 테스트 파일 및 mock
- AST 도구가 없으므로 grep 기반 검색의 한계를 인식하고,
  누락 가능성이 있으면 명시한다.

---

## 도구 결과 절삭 인식

- 도구 결과가 일정 크기를 초과하면 조용히 잘릴 수 있다.
- 검색이나 명령 결과가 의심스럽게 적으면
  범위를 좁혀(단일 디렉터리, 엄격한 glob) 재실행한다.
- 절삭이 의심되면 그 사실을 명시한다.
