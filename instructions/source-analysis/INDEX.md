# 소스 분석 하네스 INDEX

이 문서는 대규모 코드베이스를 파일 단위, 심볼 단위로 전수 분석할 때 적용하는 repo-local 하네스의 진입점이다.

## 목적

- 일부 파일만 읽고 추정하는 분석을 금지하고, 전체 범위를 끝까지 순회하는 작업 절차를 강제한다.
- 파일 인벤토리, 심볼 카탈로그, 서브시스템 맵, 공개 배포물 차이, coverage ledger를 재개 가능한 형태로 남긴다.
- 벤더 내부 소스, 공개 번들, 공식 문서, 기존 claim 문서를 서로 섞지 않고 증거 강도별로 분리한다.

## 적용 대상

- 대규모 소스 트리 전수 분석
- 공급자 제공 원본과 공개 배포물의 차이 비교
- 클래스/함수/메서드 역할 카탈로그화
- 내부 전용 분기, 공개 표면, 실험적 표면, 인프라 흔적 분리
- 후속 리버스 엔지니어링 또는 운영 RFC의 증거 기초 만들기

## 시작 전 필수 확인

1. `PLANS.md`에 전수 분석 범위, 비교 대상, 산출물 형식, 완료 기준이 append-only로 반영돼 있는지 확인한다.
2. 세션 기록에 현재 티켓과 coverage 완료 기준이 기록돼 있는지 확인한다.
3. 입력 트리가 실제로 몇 개 파일인지 먼저 확정한다.
4. 비교 대상이 있으면 같은 턴에서 경로와 역할을 명시한다.
5. 생성할 산출물의 최소 묶음을 먼저 확정한다.

## Coverage Contract

소스 분석 하네스는 아래 6개 축을 반드시 닫아야 한다.

1. **범위 봉인**
   - 입력 루트 경로
   - 전체 파일 수
   - 의도적 제외 항목 여부
2. **파일 카탈로그**
   - 각 파일의 역할, 서브시스템, import/export, 신호 태그
3. **심볼 카탈로그**
   - 함수, 클래스, 메서드, 타입, 상수 등 선언 심볼의 역할
   - 심볼이 없는 파일은 이유를 명시
4. **서브시스템 맵**
   - top-level 디렉터리별 책임과 규모
   - 진입점과 데이터 흐름의 요약
5. **배포물 차이**
   - 원본 소스와 공개 번들/패키지/문서의 대응 관계
   - 공개 표면과 내부 흔적 분리
6. **검증 ledger**
   - 실제 파일 수와 카탈로그 수 일치
   - 누락 파일 0건
   - 표본 재검증 결과

## 최소 산출물

- `INDEX.md`
- `ARCHITECTURE.md`
- `ANTI_PATTERNS.md`
- `VALIDATION.md`
- 세션 경로의 exhaustive 산출물
  - `FILE-INVENTORY.md`
  - `SYMBOL-CATALOG.md`
  - `SUBSYSTEM-MAP.md`
  - `SOURCE-NPM-DIFF.md` 또는 동등 diff 문서
  - `COVERAGE-LEDGER.md`
  - claim overlay 문서

## 산출물 작성 원칙

- 사람이 읽는 개요 문서와 기계적으로 exhaustive한 카탈로그를 분리한다.
- 전수 분석 문서에서는 “안 읽은 파일” 상태를 허용하지 않는다.
- 코드에 없는 해석을 선언 심볼 설명처럼 쓰지 않는다.
- 내부 artifact에서 나온 신호는 provenance 수준을 함께 표시한다.
- 공개 문서/공개 배포물/비공개 provenance artifact를 같은 강도의 근거처럼 쓰지 않는다.

## 읽는 순서

1. `ARCHITECTURE.md`
2. `ANTI_PATTERNS.md`
3. `VALIDATION.md`
4. 세션 경로의 `COVERAGE-LEDGER.md`
5. 세션 경로의 `SUBSYSTEM-MAP.md`
6. 세션 경로의 파일/심볼 카탈로그

## 설계 근거

- 이 저장소의 세션 규칙은 대형 조사 작업을 `PLANS/TICKETS/PROGRESS/RESEARCH/DECISIONS`로 분해해 재개 가능하게 유지한다.
- 기존 `instructions/research/*`는 조사 품질을 다루지만, 전수 코드 카탈로그와 coverage ledger 강제는 별도 하네스로 분리하는 편이 맞다.
