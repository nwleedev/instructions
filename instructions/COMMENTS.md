# `instructions/COMMENTS.md`

## 주석 규격

1. 코드 수정 후 관련 주석이 현재 구현과 일치하는지 반드시 확인한다.
2. 오래된 설명이나 잘못된 설명이 있는 주석은 수정하거나 제거한다.
3. 새로 추가한 코드에 필요한 설명이 빠져 있지 않은지 다시 확인한다.

## 주석 작성 기준

1. 코드 추가 또는 수정 시 주석이 필요한지 반드시 검토한다.
2. 주석은 모든 코드에 기계적으로 작성하지 말고 다음과 같은 경우에 우선 작성한다.
3. 공개 API, 외부 인터페이스, 라이브러리 진입점, 테스트 코드
4. 복잡한 조건 분기 또는 반복 로직
5. 비직관적인 데이터 변환 또는 상태 변화
6. 성능, 보안, 동시성 또는 시스템 제약이 존재하는 코드
7. 도메인 규칙이나 비즈니스 제약이 코드에 반영된 부분

## 주석 규격

- 언어 또는 프레임워크 생태계 표준 주석 규격을 사용한다.
  - 예를 들어 TypeScript 또는 JavaScript 프로젝트에서는 기존 규칙이 없을 경우 JSDoc 형식을 사용할 수 있다.

## 주석 내용 원칙

1. 주석은 코드가 무엇을 하는지 단순 반복하지 않는다.
2. 다음 정보를 중심으로 설명한다.

- 구현 의도
- 설계상의 제약
- 입력 또는 상태에 대한 가정
- 유지보수 시 주의해야 할 사항

## 과잉 주석 방지

1. 자명한 코드에는 주석을 작성하지 않는다.
2. 단순 getter 또는 setter
3. 이름만으로 의미가 명확한 한 줄 표현식
4. 코드 자체로 의도가 명확한 선언문
