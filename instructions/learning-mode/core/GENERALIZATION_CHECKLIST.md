# 학습 모드 범용화 체크리스트

## 목적

이 문서는 `instructions/learning-mode` 하네스를 새 주제에 그대로 재사용해도 되는지, 아니면 추가 분기 규칙이나 보조 하네스가 필요한지를 빠르게 판정하기 위한 운영 체크리스트다.

기준은 현재 저장소에서 검증된 범용 주제, 초심자 분기, research-heavy 주제, 고위험 주제 검증 묶음이다.

- IT 인접 주제:
  - AI 에이전트 아이디어 조사와 설계 문서 작성
- 비IT 일반 주제:
  - 가설검정과 실험 해석
- research-heavy 비IT 주제:
  - 온실가스 Scope 1/2/3 해석
- 초심자 프로필 검증:
  - 가설검정과 실험 해석의 초급 버전

## 1. 그대로 재사용해도 되는 주제 조건

아래 항목을 대부분 만족하면 현재 `learning-mode`를 큰 구조 변경 없이 재사용할 수 있다.

- 학습 목표가 `설명을 듣는 것`이 아니라 `무언가를 설명하거나 만드는 것`으로 끝난다.
- 주제를 3~5개의 단계 학습 루프로 나눌 수 있다.
- 공식 문서, 표준, 1차 자료, 신뢰 가능한 실무 문서 중 최소 1종 이상을 확보할 수 있다.
- 학습자의 현재 수준과 오개념을 질문 몇 개로 드러낼 수 있다.
- 최종 산출물을 메모, 설계 문서, 비교표, 체크리스트, 실행 계획 중 하나로 남길 수 있다.

대표적으로 잘 맞는 주제:

- 새 도구나 플랫폼을 익혀 설계 판단까지 내려야 하는 주제
- 개념 이해와 실무 해석이 함께 필요한 주제
- 프로젝트 시작 전에 핵심 용어, 기준, 판단 프레임을 익혀야 하는 주제

## 2. `research` 하네스를 함께 써야 하는 조건

아래 항목 중 1개 이상이면 `learning-mode` 단독보다 `instructions/research`를 보조 계층으로 병행하는 편이 안전하다.

- 최신성 의존성이 크다.
- 서로 충돌하는 출처를 비교해야 한다.
- 시장 신호, 제품 비교, 벤더 선택처럼 조사량이 크다.
- 최종 산출물이 비교 분석이나 근거 평가에 크게 의존한다.

검증된 예시:

- `온실가스 Scope 1/2/3 해석`처럼 공식 표준과 현재 개정 상태를 함께 읽어야 하는 주제는 `research` 병행이 필요했다.

운영 규칙:

- 먼저 `research`로 근거와 비교 판단을 정리한다.
- 그 다음 `learning-mode`로 개념 분해, 이해도 점검, 적용 과제를 구성한다.

## 3. 추가 분기 규칙이 필요한 조건

아래 조건이면 현재 하네스를 그대로 쓰기보다 추가 규칙이나 범위 제한을 붙여야 한다.

- 의료, 법률, 투자처럼 오답 비용이 큰 고위험 주제다.
- 계산 정확도나 증명 절차 자체가 학습 목표의 핵심이다.
- 실습 환경, 데이터셋, 장비, 사내 시스템 접근이 없으면 학습이 성립하지 않는다.
- 한 세션 산출물이 아니라 수주 단위 커리큘럼과 평가 체계가 필요하다.

권장 대응:

- 고위험 주제:
  - 설명 범위와 비전문가 한계를 먼저 명시한다.
  - 개념 학습과 개인 상황 판단을 분리한다.
  - 개인화된 결론, 실행 지시, 전문 판정 대체는 제외한다.
  - 공식 출처, 최신 확인 날짜, 사람 검토 필요 조건을 남긴다.
- 계산/증명 중심 주제:
  - 해석 학습과 계산 연습을 분리한다.
- 환경 의존 주제:
  - 준비물과 접근 권한 확인을 intake에 추가한다.
- 장기 커리큘럼:
  - 세션별 목표와 누적 평가 기준을 별도 문서로 확장한다.

빠른 판정 규칙:

- `이 개념이 무엇인가`는 대체로 허용
- `내 상황에서 지금 어떻게 결정해야 하나`는 경계 모드 또는 범위 밖
- 사람이 원문과 근거를 바로 검토할 수 없으면 안전하지 않은 구성으로 본다

## 4. 학습자 수준별 분기 힌트

### 완전 초심자

- 더 작은 용어 정리와 예시가 먼저 필요하다.
- 이해도 점검은 정의 암기보다 `틀린 해석 고치기`나 `짧은 설명 다시 쓰기`가 낫다.
- 최종 산출물은 짧은 메모나 체크리스트가 적합하다.

검증된 예시:

- `가설검정과 실험 해석` 주제는 중간 난이도 버전에서는 해석 메모를 만들었지만, 초심자 버전에서는 `지금 말할 수 있는 것 / 말하면 안 되는 것` 체크메모로 산출물을 더 작게 줄이는 편이 적합했다.

### 인접 경험은 있으나 핵심 개념이 낯선 학습자

- 가장 기본값으로 적합하다.
- 현재 하네스의 `진단 -> 단계 학습 -> 적용 산출물` 구조가 잘 맞는다.
- 최종 산출물은 설계 문서, 해석 메모, 비교표처럼 프로젝트 연결형이 적합하다.

### 숙련자는 있으나 새 도메인을 배우는 학습자

- 긴 기초 설명보다 판단 기준, 반례, 경계 조건이 중요하다.
- 이해도 점검은 `왜 이 선택을 했는가`, `어떤 상황에서 실패하는가` 같은 질문이 더 유효하다.
- 최종 산출물은 의사결정 메모, 설계 검토표, 실행 계획이 적합하다.

## 5. 세션 재개 시 반드시 확인할 질문

컨텍스트 압축이나 다음 세션 재개 시 아래 질문에 답할 수 있어야 한다.

1. 현재 학습 목표와 최종 산출물은 무엇인가
2. 마지막으로 확인된 오개념이나 막힘 지점은 무엇인가
3. 이미 사용한 핵심 근거는 무엇인가
4. 산출물 초안은 어디까지 진행됐는가
5. 다음에 바로 이어야 할 가장 작은 단계는 무엇인가

이 다섯 항목이 로컬 진행 문서, 근거 문서, 샘플 학습 로그에 남아 있지 않다면 재개 안정성이 부족한 상태로 본다.

## 6. 드라이런 후 판정 질문

새 주제로 드라이런한 뒤에는 아래를 점검한다.

- 진단 질문이 실제 난이도 조정에 영향을 줬는가
- 설명이 3회 이상 루프로 나뉘었는가
- 개방형 이해도 점검이 있었는가
- 최종 산출물이 실제 다음 작업에 재사용 가능한가
- 이 주제가 기존 검증 묶음과 다른 분기 규칙을 요구했는가

새 분기 규칙이 반복적으로 나오면 `ARCHITECTURE.md` 또는 `ANTI_PATTERNS.md`로 승격한다.

## 7. 현재 기준으로 우선순위가 높은 후속 검증

1. 실제 런타임 compaction 검증
   - 문서형 재개 검증이 실제 `thread/compact/start` 흐름에서도 유지되는지 확인한다.
2. 초심자 외 다른 하한선 검증
   - 같은 주제를 완전 비전공자 또는 시간 제약이 큰 학습자로 돌렸을 때도 단계 크기가 유지되는지 확인한다.
3. 고위험 주제 경계 검증
   - 의료, 법률, 투자처럼 현재 체크리스트가 범위 제한을 요구하는 주제에서 어떤 추가 보호 규칙이 필요한지 확인한다.
4. 고위험 주제 concept-only 드라이런
   - 새 경계 규칙이 실제 대화와 산출물에서도 유지되는지 확인한다.
   - 투자 질문 프레임과 의료 정보 읽기 주제로 검증을 마쳤고, 이후에는 법률 문서 해석처럼 문서 책임이 큰 주제로 확장 검증하는 편이 좋다.
