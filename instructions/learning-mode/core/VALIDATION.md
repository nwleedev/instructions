# 학습 모드 하네스 VALIDATION

## 기본 원칙

- 완료는 `설명을 많이 했다`가 아니라 `학습 루프와 산출물 적용 흐름이 문서와 드라이런에서 검증되었다`로 판단한다.
- 하네스는 다음 세션이나 다른 Codex 인스턴스가 그대로 이어받아도 동작 절차가 보여야 한다.

## Portable Core 완료 체크리스트

- `INDEX.md`, `core/ARCHITECTURE.md`, `core/ANTI_PATTERNS.md`, `core/PORTABILITY.md`, `core/VALIDATION.md`, `core/HIGH_RISK_BOUNDARIES.md`가 존재한다.
- 반복 재사용 기준을 설명하는 체크리스트 또는 동급 문서가 있다.
- 프로젝트별 연결 지점을 채우는 adapter 템플릿 또는 동급 문서가 있다.
- 목적, 흐름, 금지 패턴, 완료 기준이 분리되어 있다.
- 코어 문서만 읽어도 무엇을 복사하고 무엇을 로컬에서 새로 만들어야 하는지 보인다.
- 코어 문서에는 현재 머신의 절대경로나 특정 세션 ID가 없다.
- 코어 문서의 검증 규칙이 현재 저장소의 샘플 개수에 고정되어 있지 않다.
- 출처와 설계 근거가 코어 문서 안에 직접 들어 있다.
- `AGENTS.md`와 `instructions/INDEX.md`에 하네스가 등록되어 있다.

## 학습 세션 완료 체크리스트

- 학습 목표와 최종 산출물이 먼저 정의되었다.
- 학습자의 현재 수준과 제약이 진단되었다.
- 설명이 단계적으로 구성되었다.
- 이해도 점검 질문 또는 적용 과제가 단계별로 있었다.
- 중간 막힘 지점과 조정 이유가 기록되었다.
- 최종 산출물이 학습 목표와 연결된다.
- 다음 학습 행동 또는 후속 질문이 남아 있다.

## 새 프로젝트 이식 후 최소 검증 기준

다른 프로젝트에 코어를 복사했다면, 아래 local evidence를 새로 만들어야 한다.

1. 진단
   - 학습자의 현재 수준, 목표, 제약이 초기에 드러나는가
2. 단계 학습
   - 질문 기반 단계 학습 루프가 최소 3회 이상 보이는가
3. 이해도 점검
   - 예/아니오가 아닌 개방형 점검이 최소 2회 이상 있는가
4. 자료 활용
   - 공식 문서, 실제 사례, 사용자 제공 자료 중 최소 1종 이상을 활용했는가
5. 산출물 적용
   - 최종적으로 실제 재사용 가능한 산출물이 남는가
6. 세션 보존
   - 압축 또는 재개 이후에도 다음 단계가 보이도록 기록이 남는가

최소 통과 산출물:

- 대표 주제 드라이런 요약 1건 이상
- 대표 주제 학습 로그 1건 이상
- 대표 주제 최종 산출물 1건 이상
- 로컬 adapter 문서 1건
- local evidence에 맞춘 근거 목록과 한계 메모

조건부 추가 검증:

- 장기 세션 재개를 주장하면: 재개 검증 1건 이상
- 런타임 compaction까지 주장하면: 런타임 검증 1건 이상
- 고위험 주제를 다루면: 경계 규칙 + 고위험 드라이런 1건 이상
- research-heavy 주제를 다루면: `research` 병행 판정 증빙 1건 이상

## 현재 저장소의 Local Evidence Pack 판정 기준

현재 저장소는 아래 검증 묶음을 local evidence로 유지한다.

- IT 인접 주제:
  - `실제 사람들이 좋아할 수 있는 웹 애플리케이션 아이디어를 조사하고 설계 문서를 작성할 수 있는 AI 에이전트 개발`
- 비IT 일반 주제:
  - `가설검정과 실험 해석`
- research-heavy 비IT 주제:
  - `기업/기관의 온실가스 배출에서 Scope 1/2/3를 구분하고 해석하는 법`
- 초심자 프로필 검증:
  - `가설검정과 실험 해석`을 완전 초심자 수준으로 다시 분해한 드라이런
- 고위험 concept-only 검증:
  - `투자 아이디어를 검토하기 전에 어떤 질문과 경계를 먼저 확인해야 하는가`
- 전문 책임형 의료 검증:
  - `의료 가이드라인이나 건강 정보를 읽을 때 어떤 질문을 먼저 확인해야 하는가`

이 local evidence pack은 아래 산출물이 있으면 유지 기준을 통과한 것으로 본다.

- 드라이런 요약 문서 6개 이상
- 단계별 학습 로그 문서 6개 이상
- 주제별 최종 산출물 1개 이상씩
- 사용한 핵심 근거 목록과 한계 메모
- 최소 1개 주제가 개발/IT 바깥에서도 재사용 가능한 학습 산출물일 것
- 최소 1개 주제는 `research` 하네스 병행 필요성을 드러낼 것
- 최소 1개 주제는 완전 초심자 기준으로 단계 크기 축소를 보여줄 것
- 최소 1개 주제는 고위험 영역에서 concept-only 경계와 사람 검토 조건을 유지할 것
- 최소 1개 주제는 의료 또는 동급 전문 책임 영역에서 자가진단/치료선택으로 미끄러지지 않을 것

## 검증 방식

### 문서 검증

- `rg --files instructions/learning-mode`로 산출물 위치를 확인한다.
- 각 문서가 목적, 흐름, 금지 패턴, 검증 기준, 출처를 포함하는지 수동 확인한다.
- 코어 문서에 절대경로와 특정 세션 ID가 없는지 확인한다.
- `INDEX.md`만 읽고도 `portable core`, `project adapter`, `local evidence pack`이 구분되는지 확인한다.

### 드라이런 검증

- 학습 로그에서 진단 질문, 단계적 설명, 이해도 점검, 적용 과제가 확인되는지 본다.
- 최종 산출물이 샘플 주제와 직접 연결되는지 본다.
- 학습자 수준이 로컬 계획 문서나 각 드라이런에 정의한 학습자 프로필에 맞게 조정되었는지 본다.
- `research` 병행이 필요한 주제를 제대로 식별하는지 본다.
- 초심자 주제가 있다면 기존 중간 난이도보다 더 작은 설명 단위와 더 단순한 산출물을 사용하는지 본다.
- 고위험 주제가 있다면 개인화된 투자/의료/법률 조언으로 미끄러지지 않는지 본다.

### 세션 검증

- 로컬 goal/active/progress/research/decision 문서가 adapter의 선언과 일치하는지 본다.
- 다음 세션에서 이어야 할 질문과 즉시 실행 단계를 찾을 수 있는지 본다.
- `core/GENERALIZATION_CHECKLIST.md`만 읽고도 새 주제를 현재 하네스로 다룰 수 있는지 빠르게 판정 가능한지 본다.
- `evidence/RESUME_VALIDATION.md` 또는 동급의 로컬 재개 검증 문서를 기준으로 목표, 오개념, 근거, 산출물 상태, 다음 단계가 복원되는지 본다.
- `evidence/RUNTIME_COMPACTION_VALIDATION.md` 또는 동급의 로컬 런타임 검증 문서를 기준으로 실제 app-server `contextCompaction` 이후 핵심 상태가 후속 턴에서 복원되는지 본다.
- `core/HIGH_RISK_BOUNDARIES.md` 기준으로 고위험 주제가 일반 튜터링처럼 다뤄지지 않도록 경계가 명시되어 있는지 본다.

## 실패 신호

- 설명은 많은데 진단 기록이 없다.
- 이해도 점검 없이 문서만 완성되었다.
- 산출물이 학습 결과가 아니라 일반 조사 결과물처럼 보인다.
- 세션 압축 후 어디서 다시 시작해야 할지 알 수 없다.
- 새 하네스를 만들었지만 discovery 문서를 갱신하지 않았다.
- 고위험 주제인데도 사람 검토, 최신 출처, 범위 제한이 빠져 있다.
- 새 프로젝트에 복사했는데도 현재 저장소의 샘플 수와 절대경로가 필수 조건처럼 남아 있다.
