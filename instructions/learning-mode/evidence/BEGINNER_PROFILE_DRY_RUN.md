# 초심자 프로필 드라이런 요약

## 드라이런 목적

`instructions/learning-mode` 하네스가 완전 초심자에게도 너무 큰 개념 덩어리를 한 번에 밀어주지 않고, 더 작은 단계와 더 단순한 산출물로 조정되는지 확인한다.

검증 주제는 기존 두 번째 드라이런과 같은 `가설검정과 실험 해석`이지만, 이번에는 통계 용어에 익숙하지 않은 초심자 프로필을 가정한다.

검증 대상은 아래 흐름이다.

- 초심자 수준 진단
- 용어를 더 작은 단위로 쪼갠 설명
- 짧은 이해도 점검
- 짧은 산출물 적용

## 가정한 학습자 프로필

- 실험 결과 보고서를 본 적은 있지만 통계 교육은 거의 받지 않았다.
- `p-value`나 `신뢰구간`이라는 단어를 들어도 뜻을 설명하지 못한다.
- 최종적으로는 한 장짜리 실험 해석 체크메모를 보고 `지금 말할 수 있는 것`과 `지금 말하면 안 되는 것`을 구분할 수 있어야 한다.

## 드라이런 결과

### 1. 진단

- 학습자의 강점:
  - 숫자 비교와 퍼센트 변화 자체는 읽을 수 있다.
  - 실험 결과가 의사결정에 쓰인다는 사실은 이해한다.
- 학습자의 약점:
  - 가설, p-value, 신뢰구간을 각각 다른 역할로 구분하지 못한다.
  - `유의하다 = 중요한 효과다`로 바로 연결한다.
  - 긴 설명보다 짧은 문장과 구체 예시가 더 필요하다.

### 2. 단계 학습

- Step 1: 질문과 주장만 먼저 잡았다.
  - `이 실험이 답하려는 질문은 무엇인가`와 `우리가 아직 모르는 것은 무엇인가`만 먼저 쓰게 했다.
  - 귀무가설/대립가설 용어보다 `효과가 없다고 가정한 비교 기준`이라는 평이한 표현으로 시작했다.
- Step 2: `p-value`를 결론이 아니라 신호로 설명했다.
  - `0.03`이라는 숫자 하나만으로 무엇을 알 수 없는지 먼저 말하게 했다.
  - `가설이 맞을 확률`처럼 말하면 틀린 해석이라는 점을 짧은 반례로 교정했다.
- Step 3: 신뢰구간과 중요성을 분리했다.
  - `효과가 조금일 수도, 꽤 클 수도 있다`는 식으로 구간 해석을 먼저 익히게 했다.
  - 마지막에는 `유의성`, `효과 크기`, `다음 행동` 세 칸짜리 체크메모를 쓰게 했다.

### 3. 이해도 점검

- 개방형 질문 1:
  - `p-value = 0.03`만 보고 지금 확실히 말할 수 없는 것은 무엇인가
- 개방형 질문 2:
  - 왜 `통계적으로 유의하다`와 `실제로 중요하다`를 같은 뜻으로 쓰면 안 되는가
- 개방형 질문 3:
  - 오늘 메모에 `아직 모른다`고 남겨야 하는 것은 무엇인가

이 질문들에 답하도록 유도하면서, 초심자가 어려운 용어 정의를 외우기보다 틀린 해석을 피하는 쪽으로 학습 목표를 좁혔다.
