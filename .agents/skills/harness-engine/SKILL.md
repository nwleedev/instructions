# Harness Engine

이 스킬의 목적은 사용자가 언급한 작업 분야에 대해 Codex가 반복적으로 참고할 수 있는 하네스 문서를 `instructions/<task_type>/*.md`에 세팅하는 것이다.

최종 산출물은 세션 `temps`가 아니라 반드시 `instructions/<task_type>/*.md`여야 한다.

이 스킬 자체의 추가 지침과 재사용 가능한 계약은 `.agents/skills/harness-engine/` 내부에 둔다.

## 언제 사용하는가

- 새 작업 분야에 대한 하네스가 아직 없을 때
- 기존 `instructions/<task_type>` 문서가 너무 약해서 실제 작업 기준으로 쓰기 어려울 때
- 특정 분야에 대해 안티패턴, 아키텍처, 검증 기준, 템플릿을 보강해야 할 때
- 사용자가 "이 분야 작업 전에 Codex가 뭘 참고해야 하는가"를 문서화하려 할 때

## 빠른 절차

1. 사용자 요청에서 목적과 작업 분야를 추출한다.
2. 기존 `instructions/<task_type>` 하네스가 있는지, `<task_type>`와 관련된 하네스가 있는지 확인한다.
3. 기존 하네스를 재사용할지, 새 하네스를 만들지 판단한다.
4. OpenAI 공식 자료를 1차 근거로 조사하고, 실제 오픈소스 예시를 보강 근거로 사용한다.
5. `instructions/<task_type>/*.md` 산출물을 작성하거나 보강한다.
6. 드라이런과 체크리스트로 하네스가 실제로 작동하는지 검증한다.
7. 세션 기록에는 조사 근거와 작업 흐름을 남기되, 최종 하네스 세팅 환경은 `instructions/<task_type>` 아래에 남긴다.

## 작업 시작 전 확인

- 루트 `AGENTS.md`, `instructions/*.md`를 읽었는지 확인한다.
- 사용자 요청만으로 범위가 닫히는지 확인한다.
- 최종 산출물이 어느 `task_type` 아래에 위치할지 결정한다.
- 이미 있는 하네스가 있다면 우선 읽고, 덮어쓰지 말고 보강 방향을 잡는다.

## 작업 분야 Intake

`references/TASK_INTAKE.md`를 읽고 아래를 먼저 판정한다.

- 어떤 `task_type`인지
- 기존 하네스를 재사용하는지
- 새 하네스를 만들어야 하는지
- 어떤 문서 묶음이 필요한지

작업 분야가 명확하지 않으면 바로 문서를 만들지 말고, 현재 정보로 보수적 판정이 가능한지 먼저 본다.

## 산출물 계약

`references/OUTPUT_CONTRACT.md`를 읽고 최종 산출물 구성을 맞춘다.

핵심 규칙:

- 최종 산출물은 `instructions/<task_type>/*.md`
- 작업 중간 메모는 세션 `temps/`에 둘 수 있음
- 새 하네스는 가능한 한 계층형 문서 묶음으로 구성
- 예시 코드는 소스 파일이 아니라 Markdown 코드 블록으로 제시

## 검증 계약

`references/VALIDATION.md`를 읽고 최소 검증을 수행한다.

최소 확인 항목:

- 하네스만 읽고도 다음 작업 절차가 보이는가
- 금지 패턴과 권장 패턴이 분리되어 있는가
- 완료 기준과 검증 방법이 있는가
- 기존 세션/압축/재개 흐름을 해치지 않는가

## 자료 조사 규칙

OpenAI 공식 자료를 1차 근거로 사용한다.

보강이 필요할 때만 아래 순서로 확장한다.

1. OpenAI 공식 문서
2. 표준 문서
3. 실제 오픈소스 코드/문서
4. 공개 이슈나 보조 자료

블로그 단독 근거로 하네스 규칙을 확정하지 않는다.

## 기존 하네스 재사용 원칙

- 이미 `instructions/frontend`, `instructions/research`처럼 충분한 하네스가 있으면 새 디렉터리를 만들지 않는다.
- 부족한 섹션만 보강한다.
- 새 하네스는 기존 하네스로는 반복적으로 커버되지 않는 경우에만 만든다.

## 금지

- `task_type` 판단 없이 바로 새 디렉터리 생성
- 최종 산출물을 `temps/`에만 남기고 종료
- 공식 문서 없이 권장 패턴 확정
- 안티패턴과 검증 규칙을 한 문서에서 뒤섞기
- 하네스 생성 전에 기존 `instructions/<task_type>`를 읽지 않기

## 참고 파일

- 산출물 기준: `references/OUTPUT_CONTRACT.md`
- 검증 기준: `references/VALIDATION.md`
